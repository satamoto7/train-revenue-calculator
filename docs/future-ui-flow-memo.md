# 今後の実装メモ（UI導線の見直し）

実際のプレイ進行に合わせて、入力導線とタブ構成を再設計するためのメモ。

## 背景

- 現在の画面構成では「いつ・どの画面で・何を入力するか」が混在しており、実プレイ中の操作順に対して負荷がある。
- 特に「企業詳細」に収益入力と株式保有入力が同居している点は、株式ラウンド終了時の一括入力フローとズレやすい。

## 改善メモ

### 1. サマリー表示の位置づけを明確化

- サマリーは情報の俯瞰に有用。
- ただし、多くの項目は全入力後に意味を持つため、「入力中に見る画面」より「結果確認画面」としての役割を強める。
- プレイヤー別に「どの会社の株を何%保有しているか」を一覧できるボードを設ける価値がある。

### 2. 「全般管理」のタブ配置を再検討

- 全般管理はゲーム開始時に最も触るが、その後は触る頻度が低い。
- 常時並列タブに置くより、初期設定フローへ寄せる（または導線を一段下げる）方針を検討する。

### 3. 「企業詳細」から株式入力を分離

- 現状の企業詳細は、
  - 収益計算
  - 各プレイヤー/自社株の保有比率入力
    を同時に扱っている。
- 株式保有率は株式ラウンド終了時に全社分をまとめて入力する運用が想定されるため、
  - 別タブ化
  - そのタイミングに合った入力導線
    への分離を検討する。

### 4. プライベートカンパニー収益の補正入力

- 各 OR ラウンドで所有者に収益が入るプライベートカンパニー向けに、オプション入力欄を設ける案。
- 一方で OR 中に閉鎖や売買が発生しうるため、固定フォームより自由入力の補正欄でも運用可能。
- 候補:
  - サマリーの「総配当」欄に補正欄を設置
  - 企業詳細でプレイヤーに紐づく補正欄を設置

## 次アクション案

1. 画面別に「利用タイミング（開始時 / 株式R後 / OR中 / 集計時）」を整理する。
2. 既存タブを「設定」「株式」「運転（OR）」「集計」に再マッピングしたワイヤーを作成する。
3. 株式入力を一括編集できる簡易プロトタイプを先に実装し、入力負荷を検証する。
